# NixOS Configuration Documentation for host: nixos-ap

This document provides an overview of the NixOS configuration for the `nixos-ap` host, managed via a Nix Flake.

## File Structure

The configuration is organized into the following main files and directories:

- **`flake.nix`**: The entry point of the configuration. It defines the inputs (like `nixpkgs` and `home-manager`) and the outputs (the NixOS system configuration).

- **`hosts/nixos-ap/`**: This directory contains the host-specific configuration.
  - **`configuration.nix`**: The main configuration file for `nixos-ap`. It imports all the necessary modules.
  - **`hardware-configuration.nix`**: Hardware-specific settings, automatically generated by NixOS during installation.

- **`modules/`**: This directory contains modularized parts of the configuration.
  - **`desktop.nix`**: Configures the GNOME desktop environment, including excluding default applications.
  - **`packages.nix`**: Manages system-wide packages.
  - **`dev.nix`**, **`services.nix`**, **`system.nix`**, **`user.nix`**: Other modularized configuration files.

- **`home-manager/`**: This directory contains the `home-manager` configuration.
  - **`home.nix`**: Defines user-specific packages, dotfiles, and services.

## Package Management

Packages are managed in two ways:

1.  **System-wide packages**: Defined in `modules/packages.nix` and `modules/desktop.nix` (via `environment.systemPackages`). These packages are available to all users.
2.  **User-specific packages**: Defined in `home-manager/home.nix` using `home.packages`. These packages are only available to the specified user (`abhishek`).
